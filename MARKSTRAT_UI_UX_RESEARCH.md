# Markstrat UI/UX 레퍼런스 조사 보고서

> BizSim 디자인 개선을 위한 Markstrat & 유사 경영 시뮬레이션 게임 분석

---

## 1. Markstrat UI/UX 특징 정리

### 1.1 레이아웃 및 구조

**전반적 레이아웃**
- **대시보드 형식**: 좌측 사이드바 + 상단 헤더 + 중앙 메인 콘텐츠 영역
- **사이드바 구성**
  - 게임 진행 단계(Phase) 표시
  - 의사결정 메뉴 (마케팅, R&D, 생산 등)
  - 리포트/분석 메뉴
  - 메뉴 선택 시 중앙 콘텐츠 동적 변경
  
- **상단 헤더**
  - 현재 라운드 번호
  - 팀/회사 이름
  - 알림/메시지
  - 설정/로그아웃

- **메인 콘텐츠 영역**
  - 의사결정 입력 양식
  - 데이터 시각화 (차트, 테이블)
  - 시장 정보/리포트
  - 동적 콘텐츠 로딩

**패널/섹션 구성**
- 여러 개의 카드/패널로 분할
- 각 패널 = 하나의 의사결정 영역 (예: 광고 예산, R&D 투자, 가격 설정)
- 패널 내부에 입력 필드, 슬라이더, 미니 차트 배치
- 카드 기반 레이아웃으로 시각적 계층 표현

**대시보드 레이아웃**
- 핵심 성과 지표(KPI) 강조 표시
  - 매출, 시장 점유율, 이익, 브랜드 인지도 등
  - 큰 폰트 + 강조색 (보통 파란색 또는 초록색)
- 요약 통계와 상세 보기를 분리

### 1.2 색상 팔레트 및 시각적 스타일

**주요 색상**

| 요소 | 추정 색상 코드 | 설명 |
|------|---------------|------|
| 주요 강조색 | `#007bff` (Bright Blue) | 버튼, 링크, 활성 상태 |
| 배경색 | `#f8f9fa` (Light Gray) 또는 흰색 | 콘텐츠 배경 |
| 텍스트색 | `#343a40` (Dark Gray) | 본문 텍스트 |
| 보조 강조색 | `#28a745` (Green) | 긍정적 변화, 이익 |
| 경고색 | `#dc3545` (Red) | 손실, 경고 |
| 중립색 | `#6c757d` (Medium Gray) | 비활성, 라벨 |

**테마**
- **기본**: Light 테마 (흰색/밝은 회색 배경)
- **특징**: 깔끔하고 정돈된 비즈니스 소프트웨어 느낌
- **다크 테마**: 드물거나 미지원

**시각적 스타일**
- **미니멀리즘**: 불필요한 장식 제거
- **기능성 중심**: 정보 전달에 최우선
- **명확성**: 아이콘과 텍스트로 기능 표현
- **일관성**: 전체 인터페이스에 같은 색상 + 스타일 적용

**폰트**
- **주요 폰트**: Sans-serif 계열
  - Arial, Helvetica, Lato, Roboto 등 (추정)
- **크기 계층**
  - 제목: 18-24px
  - 본문: 12-14px
  - 라벨: 11-12px
  - 캡션: 10-11px

### 1.3 의사결정 입력 방식

**슬라이더 (Range Slider)**
- **사용처**: 예산 배분, 가격 설정, 광고 분배
- **특징**:
  - 연속적인 값 입력 가능
  - 슬라이더 이동 중 현재 값 실시간 표시
  - 최소/최대값 경계 설정
  - 숫자 입력 필드와 병행

**폼 (Form)**
- **사용처**: 신제품 출시, 세그먼트 선택, 채널 선택
- **컴포넌트**:
  - 텍스트 입력 필드 (제품명 등)
  - 드롭다운/셀렉트 (세그먼트, 채널 선택)
  - 체크박스 (기능 추가/제거)
  - 라디오 버튼 (배타적 선택)
  - 텍스트 에어리어 (설명)

**테이블 (Grid/Table)**
- **사용처**: 생산량 계획, 채널 할당, 인력 배치
- **특징**:
  - Excel과 유사한 셀 기반 입력
  - 행/열 합계 자동 계산
  - 대량 데이터 빠른 입력 가능
  - 정렬/필터 기능

**버튼 배치**
- **배치 위치**: 화면 하단 또는 패널 우측
- **주요 버튼**:
  - "의사결정 제출" (Submit/Confirm) - 강조색
  - "다음 라운드" (Next Round) - 보조색
  - "저장" (Save) - 중립색
  - "취소" (Cancel) - 회색

**실시간 피드백**
- 입력값 초과 경고 (예: 예산 한도 초과)
- 유효하지 않은 입력 표시 (빨간색 경계선)
- 오류 메시지 (인라인, 상단 알림)
- 성공 메시지 (초록색 배경)

## 2. 주요 인터랙션 패턴

### 2.1 의사결정 입력 워크플로우

```
1. 메뉴 선택
   ↓
2. 데이터 검토 (시장 보고서, 경쟁사 정보)
   ↓
3. 값 입력/조정 (슬라이더, 폼, 테이블)
   ↓
4. 효과 예측 (선택사항: 실시간 미리보기)
   ↓
5. 의사결정 저장/제출
```

**각 단계의 특징**

| 단계 | UI 요소 | 피드백 |
|------|---------|--------|
| 메뉴 선택 | 사이드바 + 탭 | 선택 항목 강조 |
| 데이터 검토 | 리포트 패널 + 차트 | 차트 호버 시 상세정보 |
| 값 입력 | 슬라이더, 입력 필드 | 실시간 값 업데이트 |
| 효과 예측 | 예상 결과 차트 | 현재값과 예상값 비교 |
| 저장/제출 | 버튼 + 확인 모달 | 성공/실패 메시지 |

### 2.2 라운드 진행 흐름

```
라운드 시작
  ↓
이전 라운드 결과 요약 표시
  ↓
시장 보고서 분석
  ↓
의사결정 수행 (마케팅, R&D, 생산, 가격 등)
  ↓
의사결정 제출 (팀의 모든 의사결정 완료 시)
  ↓
[백엔드 시뮬레이션]
  ↓
다음 라운드 결과 표시 (대시보드 + 상세 리포트)
```

### 2.3 내비게이션 패턴

- **사이드바 네비게이션**: 주 메뉴 (항상 표시)
- **탭 네비게이션**: 서브 메뉴 또는 콘텐츠 전환
- **breadcrumb**: 깊은 계층 표현 (선택사항)
- **버튼/링크**: 의사결정 저장 후 다음 화면으로 이동

## 3. 시각화 스타일

### 3.1 차트 종류 및 활용

**라인 차트 (Line Chart)**
- **사용처**: 시간별 추세 표현
- **데이터**: 시장 점유율, 매출, 이익, 브랜드 인지도 변화
- **특징**: 여러 라인을 동시에 표현 가능 (자사, 경쟁사 비교)
- **라이브러리**: Recharts, Chart.js, Plotly

**바 차트 (Bar Chart)**
- **사용처**: 범주별 비교
- **데이터**: 세그먼트별 매출, 경쟁사별 시장점유율, 예산 배분
- **특징**: 
  - 수평/수직 바
  - 스택형 바 (구성요소 비교)
  - 그룹화된 바 (다중 변수 비교)

**파이/도넛 차트 (Pie/Donut Chart)**
- **사용처**: 비율/구성 표현
- **데이터**: 세그먼트 시장 비중, 예산 구성, 채널별 매출
- **특징**: 전체 대비 부분 비중 직관적 표현

**산점도 (Scatter Plot)**
- **사용처**: 제품 포지셔닝 맵 (Perceptual Map)
- **축**: X축 (가격 또는 품질), Y축 (브랜드 인지도 또는 만족도)
- **마크**: 각 제품을 점으로 표현, 크기는 판매량 또는 시장점유율
- **매우 중요**: Markstrat의 핵심 시각화

**영역 차트 (Area Chart)**
- **사용처**: 시간별 누적 값 표현
- **데이터**: 채널별 누적 매출, 세그먼트별 누적 점유율
- **특징**: 구성 요소별로 색상 구분

**히트맵 (Heatmap)**
- **사용처**: 다차원 데이터 비교 (선택사항)
- **데이터**: 세그먼트별 성과 매트릭스

### 3.2 그래프 스타일 가이드

**시각 계층**
- **중요도**: 크기, 색상 명도 차등 적용
- **대비**: 배경과 데이터 충분한 대비
- **일관성**: 같은 의미의 데이터는 같은 색상

**인터랙션**
- **호버**: 특정 데이터 포인트 선택 시 상세 정보 툴팁
- **범례**: 범례 클릭으로 데이터 시리즈 표시/숨김
- **확대/축소**: 드래그로 영역 확대, 리셋 버튼으로 원래대로

**색상 전략**
- **색상표**: 전체 게임에서 일관된 색상 사용
  - 자사 = 파란색
  - 경쟁사 A = 주황색
  - 경쟁사 B = 빨간색
  - 시장 평균 = 회색
  - 긍정/이익 = 초록색
  - 부정/손실 = 빨간색

**데이터 표현**
- **숫자 + 시각**: 차트 위에 수치 라벨 표시
- **변화율**: % 또는 화살표(↑/↓) 표시
- **포맷팅**: 통화($), 백분율(%), 천 단위 구분 기호

## 4. 유사 경영 시뮬레이션 게임 UI 비교

**컴포넌트**
- ✅ 슬라이더 (연속값 입력)
- ✅ 드롭다운/셀렉트
- ✅ 테이블 (데이터 입력/표시)

## 5. BizSim 적용 권장사항

### 5.1 레이아웃

- **사이드바 네비게이션** 도입
  ```
  ├─ Dashboard (대시보드)
  ├─ Decisions (의사결정)
  │  ├─ Marketing Mix
  │  ├─ R&D & Product
  │  ├─ Production
  │  └─ Pricing
  ├─ Reports (리포트)
  │  ├─ Market Analysis
  │  ├─ Financial Statements
  │  └─ Competitor Analysis
  └─ Settings
  ```

- **카드형 패널** 구조
  - 각 의사결정 영역을 별도 카드로 표현
  - 패널 내 슬라이더 + 미니 차트 병행

- **대시보드**: 핵심 KPI를 큰 카드로 강조

### 5.2 색상 시스템

```css
/* Markstrat 레퍼런스 색상 */
--color-primary: #007bff;      /* 강조색 (버튼, 링크) */
--color-success: #28a745;      /* 긍정/이익 */
--color-danger: #dc3545;       /* 손실/경고 */
--color-warning: #ffc107;      /* 주의 */
--color-bg-light: #f8f9fa;     /* 밝은 배경 */
--color-bg-white: #ffffff;     /* 콘텐츠 배경 */
--color-text-dark: #343a40;    /* 본문 텍스트 */
--color-text-muted: #6c757d;   /* 약한 텍스트 */
```

### 5.3 차트 및 시각화

- **필수 차트**:
  1. **대시보드**: 라인 차트 (시계열 KPI)
  2. **시장 분석**: 산점도 (포지셔닝 맵)
  3. **경쟁사 분석**: 바 차트 (시장점유율)
  4. **재무 보고서**: 라인 차트 (매출/이익) + 바 차트 (구성)

- **라이브러리**: Recharts (현재 BizSim에서 사용 중)

- **상호작용**:
  - 호버 시 상세정보 표시
  - 범례 클릭으로 데이터 시리즈 토글
  - 확대/축소 가능

### 5.4 컴포넌트 및 인터랙션

**입력 패턴**
- 예산 배분: 슬라이더 + 숫자 입력
- 제품 특징: 멀티 셀렉트
- 가격 설정: 슬라이더 또는 입력 필드
- 채널 할당: 테이블 (행=채널, 열=예산)

**피드백**
- 입력값 초과 시 즉시 경고 (빨간색 경계선)
- 의사결정 제출 전 검증 (누락 항목 표시)
- 저장 성공 시 초록색 토스트 메시지

**라운드 진행**
- "라운드 시작" 버튼: 결과 로딩
- 진행률 표시 (예: "Round 3/10")
- "의사결정 제출" 버튼: 모든 필드 검증 후 활성화
- 다음 라운드로 자동 전환 또는 "계속" 버튼

### 5.5 폰트 및 타이포그래피

```css
/* Sans-serif 기반 */
font-family: "Roboto", "Lato", "Helvetica", sans-serif;

/* 크기 스케일 */
--text-xs: 11px;      /* 캡션 */
--text-sm: 12px;      /* 라벨 */
--text-base: 14px;    /* 본문 */
--text-lg: 16px;      /* 서브헤드 */
--text-xl: 18px;      /* 섹션 제목 */
--text-2xl: 24px;     /* 페이지 제목 */
--text-3xl: 32px;     /* 메인 제목 */

/* 가중치 */
font-weight: 400;     /* Regular (본문) */
font-weight: 600;     /* Semi-bold (제목) */
font-weight: 700;     /* Bold (강조) */
```
